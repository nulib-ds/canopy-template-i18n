## Bienvenida

Canopy renderiza cada idioma a partir de los mismos datos IIIF y la misma estructura de carpetas. El inglés vive directamente bajo `content/`, mientras que los demás idiomas (como `es`) viven en `content/<lang>/`. Cuando las rutas y los nombres de archivo coinciden, el selector de idioma puede descubrir qué versión mostrar.

### Extiende los locales

1. Edita `canopy.yml` para declarar los idiomas en `locales` y marca `default: true` para el idioma principal.
2. Usa códigos BCP 47 (por ejemplo `es` o `pt-BR`) y etiquetas legibles.
3. Reinicia `npm run dev` o vuelve a ejecutar `npm run build` para aplicar los cambios.

### Refleja el contenido

- Traduce los archivos MDX en `content/<lang>/…` usando los mismos nombres (`content/index.mdx` ⇄ `content/es/index.mdx`).
- Si falta una traducción, Canopy usa el contenido del idioma predeterminado, así que puedes traducir gradualmente.
- Usa frontmatter para fijar títulos o `lang` propios.

### Localiza la interfaz

- Copia `content/locale.yml` hacia `content/<lang>/locale.yml` y traduce los textos.
- Ajusta `routes.search` y `routes.works` para cambiar las rutas (`/es/buscar`, `/es/obras`).
- Traduce el menú en `content/<lang>/navigation.yml`.
